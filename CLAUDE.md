# Main
- Никогда не выдумывай инфомарцию, всегда говори что не знаешь или спрашивай!
- Для получения документации и инофрмации о библиотеках используй mcp context7

# Docker
- всегда используй docker compose вместо docker-compose
- если нужно установить зависимости, использовать много зависимостей всегда используй docker контейнеры

# Git
- не надо подписывать git коммиты как Co-Authored-By и Generated with 
- Коммиты не надо супер сильно рассписывать, не более 15 строк
- Коммиты писать только на английском языке
- нельзя использовать смайлы и специальные символы в текте коммитов
- нельзя никогда делать git reset в любом виде
- нужно делать атомарные коммиты, т.е. каждый коммит должен решать одну задачу

# Стиль кодирования
- Не надо заранее писать код, который не будет использоваться в текущей задаче
- используй стиль KISS

# Документация
- Документация должна быть написана в md формате
- Документация должна быть написана на русском и английском языке
- вся документация должна лежать в папке docs, кроме README.md

# Утилиты
- для http запросов можно использовать curl, wget
- для тестов страниц и дизайна используй mcp playwright
